## рҹ’ҫ OVERVIEW рҹ’ҫ

**Welcome to the time machine!** рҹ•°пёҸвңЁ 

Remember when Visual Basic 6 was the *king* of RAD (Rapid Application Development)? When you could drag-drop an ActiveX control and ship enterprise software faster than dial-up could connect? When `On Error Resume Next` was... well, let's not talk about that. рҹҳ…

This lab takes you on a **radical journey** from the golden age of Windows 98 to the modern cloud era! You'll modernize a **real-world VB6 manufacturing application** that's been running shop floor operations since 2001. We're talking MSFlexGrid, ADO Recordsets, Crystal Reports, and enough global variables to make your head spin! рҹҢӘпёҸ

**Business Domain:** Precision Parts Manufacturing рҹҸӯ  
A small manufacturing company's inventory and work order tracking system. It manages raw materials, production orders, quality inspections, and shipping вҖ” all from a trusty VB6 desktop app connected to an Access database on a network share. Classic! рҹ’Ә

---

## рҹҺҜ WHAT YOU'LL LEARN рҹҺҜ

рҹ”Ҙ **Assess** VB6 applications for modernization scope and complexity  
рҹ”Ҙ **Migrate** Microsoft Access databases to SQL Server / Azure SQL  
рҹ”Ҙ **Translate** VB6 forms and business logic to modern C# patterns  
рҹ”Ҙ **Eliminate** global state with dependency injection (goodbye `modGlobals.bas`!)  
рҹ”Ҙ **Choose** between desktop (WPF) and web (Blazor) modernization targets

рҹ”Ҙ **Transform** Crystal Reports into QuestPDF masterpieces  
рҹ”Ҙ **Deploy** to Azure App Service (Blazor) or MSIX packages (WPF)

## рҹҸӣпёҸ LEGACY STACK рҹҸӣпёҸ

### рҹ’ҝ THE CLASSICS рҹ’ҝ

**Language & Runtime:**
- рҹҺ№ Visual Basic 6 SP6
- рҹӘҹ Windows Common Controls 6.0
- рҹ”Ң ActiveX Components (OCX files galore!)

**Data Access Layer:**
- рҹ“Ў **ADO 2.8** (ADODB.Connection, ADODB.Recordset)
- рҹ’ҫ **Microsoft Access 2003** (MDB file on network share)
- рҹ”җ File-locking for concurrent access (oh boy!)

**UI Controls:**
- рҹ“Ҡ **MSFlexGrid** вҖ” The grid to rule them all
- рҹ—ӮпёҸ **CommonDialog** вҖ” File picker supreme
- рҹ–ұпёҸ **Windows Common Controls** вҖ” Toolbar, StatusBar, TreeView

**Reporting:**
- рҹ“„ **Crystal Reports 8.5** (unsupported vintage edition)

**System Integration:**
- рҹ§° **Windows API Calls** (user32.dll, kernel32.dll)
- рҹ“қ **Registry Settings** (SaveSetting/GetSetting)

**Database Schema:**
```
Tables: Parts, RawMaterials, WorkOrders, QualityChecks, 
        ShippingManifests, Customers, Users
```

### вҡ пёҸ BLUE SCREEN рҹ’ҷ вҖ” LEGACY ANTI-PATTERNS DETECTED!

рҹҡЁ **Global Variable Overload** вҖ” `modGlobals.bas` with 50+ public variables  
рҹҡЁ **GoTo Error Handling** вҖ” `On Error GoTo ErrHandler` everywhere  
рҹҡЁ **Network Share Database** вҖ” Access MDB with file locking  
рҹҡЁ **Plain Text Passwords** вҖ” Yes, really  
рҹҡЁ **Hardcoded Paths** вҖ” File paths and connection strings everywhere  
рҹҡЁ **DoEvents Hell** вҖ” Keeping UI responsive the old-school way  
рҹҡЁ **No Unit Tests** вҖ” Testing was for the weak! (jk, please test)
